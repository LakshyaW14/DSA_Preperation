# 🔥Dynamic Programming (DP) ->

---

### 1. Simple definition (one sentence)

A technique where **we solve complex problems by breaking them into overlapping subproblems and storing results to avoid recomputation**.

---

### 2. Real-world analogy

Think of **calculating Fibonacci numbers**—instead of recalculating the same values again and again, you **remember (store) previous results** and reuse them.

---

### 3. When to use it (pattern/situation)

Use DP when you see:

* **Overlapping subproblems**
* **Optimal substructure**
* Keywords like:

  * “maximum / minimum”
  * “count ways”
  * “best / optimal”
  * “can we reach?”

👉 Common patterns:

* **1D DP** (Fibonacci, climbing stairs)
* **2D DP** (knapsack, LCS)
* **DP on strings**
* **DP on grids**

---

### 4. Basic code template

#### Python (Top-Down / Memoization)

```python id="1c6lqk"
def dp(i):
    if base_case:
        return value
    
    if i in memo:
        return memo[i]
    
    memo[i] = recurrence_relation
    return memo[i]

memo = {}
```

#### Python (Bottom-Up / Tabulation)

```python id="m6t7kq"
def dp(n):
    dp = [0] * (n + 1)
    dp[0] = base
    
    for i in range(1, n + 1):
        dp[i] = recurrence_relation
    
    return dp[n]
```

👉 **Time Complexity:** Usually O(n) / O(n²)
👉 **Space Complexity:** O(n) (can optimize sometimes)

### 5. Easy example + walkthrough

**Problem:** Climbing Stairs
You can take 1 or 2 steps → how many ways to reach n?

#### Code

```python id="6eqr2p"
def climb(n):
    if n <= 2:
        return n
    
    dp = [0] * (n + 1)
    dp[1], dp[2] = 1, 2
    
    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

#### Walkthrough

* dp[1] = 1
* dp[2] = 2
* dp[3] = 3
* dp[4] = 5

👉 Same as Fibonacci pattern

### 6. Common mistakes beginners make

* ❌ Not identifying overlapping subproblems
* ❌ Wrong state definition
* ❌ Incorrect recurrence relation
* ❌ Missing base cases
* ❌ Using DP where greedy works better
